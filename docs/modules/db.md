# Unified DB (`BiznizDB`)

`bizniz/db/`. The unified database used when `BiznizConfig.database_url` is set.

## Why it exists

Originally each project had its own SQLite file (`.bizniz/project.db`) and each workspace had its own SQLite file (`.bizniz/bizniz.db`). For production deployments, that's clumsy â€” you can't run analytics across projects, you can't easily back things up, and remote workers can't share state.

`BiznizDB` puts everything in one MySQL or SQLite database and uses two scoped accessors (`ProjectScope` and `WorkspaceScope`) that automatically inject the correct `project_id` / `service_name` filters into every query.

## Files

| File | Class | Purpose |
|------|-------|---------|
| `bizniz_db.py` | `BiznizDB` | Connection manager, schema DDL, scope factories |
| `project_scope.py` | `ProjectScope` | Drop-in replacement for `ProjectDB` â€” same API, scoped to one `project_id` |
| `workspace_scope.py` | `WorkspaceScope` | Drop-in replacement for `WorkspaceDB` â€” same API, scoped to `(project_id, service_name)` |

## `BiznizDB`

Constructor: `BiznizDB(database_url="sqlite:///:memory:")`. Routes by URL scheme:

- `mysql://...` â†’ uses `pymysql` with DictCursor; autocommit off.
- Anything else (including `sqlite:///`, `:memory:`, or a bare path) â†’ `sqlite3` with `Row` factory and FK pragma on.

Public surface:

| Method | Returns | Notes |
|--------|---------|-------|
| `for_project(project_id)` | `ProjectScope` | Project-level accessor |
| `for_workspace(project_id, service_name)` | `WorkspaceScope` | Workspace-level accessor |
| `backend` | `"mysql" \| "sqlite"` | Read-only |
| `close()` | None | Commits and closes the connection |

Private helpers:

- `_execute(sql, params)` automatically converts `?` placeholders to `%s` for MySQL.
- `_upsert_sql(table, columns, conflict_cols, update_cols)` produces backend-specific upsert DDL (`ON CONFLICT` for SQLite, `ON DUPLICATE KEY UPDATE` for MySQL).

## Schema

`_create_tables()` runs an array of `CREATE TABLE IF NOT EXISTS` DDL on construction. Tables fall into two scopes:

### Workspace-scoped (require `project_id` + `service_name`)

| Table | Purpose |
|-------|---------|
| `problems` | Problem statements submitted to the engineer |
| `requirements` | business / functional / nonfunctional |
| `use_cases` | Title + description per use case |
| `issues` | Coding tasks: title, description, status (`open`/`in_progress`/`closed`), target_files_json, test_files_json, depends_on_json, suggested_model, test_setup_hint |
| `architecture_plans` | Per-problem plan, versioned, with full plan_json |
| `architecture_namespaces` | `(plan_id, namespace_path)` unique |
| `architecture_domain_models` | Class + filepath + definition_json + namespace |
| `architecture_modules` | Filepath + class_name + namespace + definition_json. Unique on `(plan_id, filepath, class_name)`. |
| `architecture_dependencies` | Source/target filepath + import symbols |
| `test_results` | Per-issue, per-test-file pass/fail tracking |
| `environment_packages` | Pip packages installed in the runner container, per-workspace |
| `environment_config` | Free-form key/value config |

### Project-scoped (require `project_id`)

| Table | Purpose |
|-------|---------|
| `services` | Service registry + image tag + status |
| `architecture_snapshots` | Full `SystemArchitecture` JSON, versioned |
| `issue_log` | Cross-service issue ledger |
| `build_log` | `image_build` / `package_install` / `rebuild` events |
| `drift_events` | Unplanned files generated by the coder |

## Example

```python
from bizniz.db.bizniz_db import BiznizDB

db = BiznizDB("mysql://user:pass@localhost/bizniz")

proj = db.for_project("pet_groomer")
proj.save_service(
    name="backend", service_type="backend",
    framework="fastapi", language="python",
    workspace_path="/projects/pet_groomer/backend",
)

ws = db.for_workspace("pet_groomer", "backend")
problem_id = ws.save_problem("Build endpoints for grooming appointments")
issue_id = ws.save_issue(
    problem_id=problem_id,
    title="POST /appointments",
    description="...",
    target_files=[{"filepath": "api/appointments.py", "action": "create"}],
)
db.close()
```

## Interactions

- **Calls into:** `pymysql` (MySQL backend) or `sqlite3` (SQLite backend).
- **Used by:** `BiznizConfig.make_db()`, `BaseWorkspace.db` (when `bizniz_db` is provided), `Project.db` (when `bizniz_db` is provided).

## Gotchas

- **Two backends, one query language.** `?` placeholders only â€” `_execute` converts to `%s` for MySQL. Don't write provider-specific SQL by hand.
- **Upserts have hand-rolled DDL.** `_upsert_sql` constructs the right backend-specific clause. If you add a unique index, both branches need the matching syntax.
- **`for_workspace` and `for_project` create fresh scope objects every call.** They're cheap; don't cache them yourself.
- **MySQL is `autocommit=False`.** Every write goes through `_commit()` after the cursor's `execute()`. Forgetting `_commit()` silently drops the change.
- **The DB is shared across services in unified mode.** A misconfigured `service_name` would let one service read/write another's data. The scope wrappers always include both filters in `WHERE`, so the only way to bleed across is if you handcraft SQL via `_execute(...)` with the wrong values.
- **Standalone mode is still the default.** Without `BIZNIZ_DATABASE_URL` (or `database_url` in YAML), the older `WorkspaceDB` / `ProjectDB` are used. The unified scope objects are drop-in replacements with the same method names so callers don't notice.
